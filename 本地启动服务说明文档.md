# WebPerf Scout 本地启动指南
项目: "具备爬虫能力的web软件性能分析与测评工具"

## 1. 环境准备

确保本地环境已安装以下工具：

- **Node.js** (版本 18+)
  - 下载地址：[Node.js 官网](https://nodejs.org/)
- **npm** 或 **yarn** (推荐使用 yarn)
  - 安装命令：`npm install -g yarn`

## 2. 项目结构检查

确保您的项目目录结构如下：

```
webperf-scout-161021/
├── frontend/          # 前端代码目录
│   ├── package.json   # 前端依赖文件
│   └── src/           # 前端源代码
└── backend/           # 后端代码目录
    ├── package.json   # 后端依赖文件
    └── server.js      # 后端入口文件
```

## 3. 安装依赖

分别进入前端和后端目录，安装依赖：

### 安装前端依赖
```bash
# 进入前端目录
cd frontend
yarn install  # 或 npm install
```

### 安装后端依赖
```bash
# 进入后端目录
cd ../backend
yarn install  # 或 npm install
```

## 4. 启动后端服务

后端服务默认使用 `server.js` 作为入口文件，启动命令如下：

```bash
# 确保位于 backend 目录
cd backend
node server.js
```

后端默认运行在 **http://localhost:3000**（具体端口请查看 `server.js` 中的配置）。

## 5. 启动前端服务

前端服务默认使用 React 开发服务器，启动命令如下：

```bash
# 确保位于 frontend 目录
cd frontend
yarn dev  # 或 npm run dev
```

前端默认运行在 **http://localhost:5173**（Vite 默认端口），并自动打开浏览器。

## 6. 验证服务

1. 打开浏览器访问 **http://localhost:5173**，确保前端页面正常加载。
2. 检查后端接口是否可用（如 **http://localhost:3000/api/start-task**）。

## 注意事项

- 请确保先启动后端服务，再启动前端服务
- 如果端口被占用，请修改相应的配置文件中的端口号
- 开发环境下，前端会自动代理 API 请求到后端服务

## 故障排除
如果遇到问题，请检查：
- Node.js 版本是否符合要求
- 依赖是否安装完整
- 端口是否被其他程序占用
- 防火墙设置是否允许本地服务访问
